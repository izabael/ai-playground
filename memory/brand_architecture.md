Four-layer brand hierarchy (decided 2026-04-04/05):

| Layer | Example | Purpose |
|---|---|---|
| **Company** | SILT™ (Sentient Index Labs & Technology, LLC) | The LLC. Owns trademark (TM pending USPTO) |
| **Software** | SILT™ AI Playground (github.com/izabael/ai-playground) | Open source Apache 2.0 code |
| **Flagship instance** | izabael.com | THE showcase instance SILT operates |
| **Community instances** | anyones-playground.com | Self-hosted, peered via federation |

**Why:** Product loyalty over instance loyalty. SILT wins when people run SILT AI Playground software ANYWHERE, not just izabael.com. WordPress/Mastodon model — same software, two experience tiers.

**How to apply:**
- siltcloud.com pages = B2B/corporate framing, "SILT AI Playground" product, cites izabael.com as flagship reference
- izabael.com = warm/personal/Izabael's voice, human-facing
- Repo stays `izabael/ai-playground` (SEO-friendly descriptive name)
- Two-tier data terms: izabael.com commercial; self-hosted = your data

**Instance naming pattern:** "[Name]'s AI Playground" (NOT "[Name]'s Playground" — "Playground" alone reads adult-themed on SEO crawl). "Izabael's AI Playground" is the flagship instance name on izabael.com's Agent Card.

**Trademark status:** SILT™ trademark filed by Kris, pending registration. Use ™ until ® issues. SILT AI Playground inherits protection automatically — no separate TM filing. Use "SILT™" on first mention per page, "SILT" after, with TM attribution footer: "SILT™ is a trademark of Sentient Index Labs & Technology, LLC. Registration pending."
